02-06-26 - 02-18-26 Setting Up the Architecture
- This is a diagram of the new structure of my lab infrastructure with the various phases and software involved.
- I began this phase by restructuring how I approach building the lab. Previously, I ran into several issues because Hyper-V was installed and configured in a way that caused problems later on. After restarting from scratch, I focused on organizing my files properly and documenting each step so I could avoid repeating those mistakes.
Instead of manually installing Hyper-V through scripts like I attempted before, I enabled it directly through Windows Features. This made the setup process significantly cleaner and more stable.
- To keep everything organized, I created a dedicated folder structure inside my Home-Labs directory. Within the Hyper-V Files folder, I separated my resources into three main folders: ISOs, VMs, and VHDs. This allows me to clearly manage installation media, virtual machines, and virtual hard disks without confusion as the lab grows.
- The next step was obtaining the Ubuntu 24.04.3 Server ISO. Ubuntu will serve as the internal file/application server in this first phase. My immediate goal with Ubuntu is to install Apache and host a file named “Salmiakki.png” inside the web directory. Being able to access this file internally will confirm that the network is functioning as intended.
- After that, I downloaded the pfSense ISO from Netgate and extracted it for later use. At this stage, Hyper-V, Ubuntu, and pfSense are the only components required to build the foundational network. The plan is to configure an Internal Virtual Switch so that I can directly access the Ubuntu server from my host machine while still maintaining network segmentation through pfSense.
- Once the internal connectivity is confirmed and I can retrieve the hosted image successfully, future phases will focus on testing and security validation. This will include introducing a separate Windows virtual machine, using tools such as Nmap to test for weaknesses, and determining whether segmentation rules properly restrict access when intended.
- After the infrastructure is stable, I plan to take Splunk’s free “Intro to Splunk” and “Using Fields” courses before implementing the SIEM layer into the environment.
